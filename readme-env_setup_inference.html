Environment Setup for Inference Moduleïƒ

Overviewïƒ

This document covers the environment setup for the inference module serving the fine-tuned WavLM model.

  • For inference, you can choose to use pre-built Docker image or serve from the source repository .

  • Both methods require installation of Docker and Docker-compose. Please ensure you have these installed before proceeding.

Inference: Method 1 - Deploy from pre-built Docker imageïƒ

Download and unzip the file klass2-final-deployment-inference.zip, and ensure files are in the following directory structure, creating all the directories as needed:

Set directory permissionsïƒ

Our docker-compose.yml specifies that the docker images are running as non-root users (specifically, user osdklass, user id 2222, group osdklass, group id 2222), for better security.

Thus you will need to set permissions for the vol_mount directory so that the containers are able to access them properly:

# Create a user:group osdklass:osdklass (UID 2222:GID 2222)
# to match the user:group in the Docker containers
$ sudo groupadd -g 2222 osdklass
$ sudo useradd -u 2222 -g 2222 -m osdklass

# Set directory permissions to 770 (read, write, and execute for owner and group)
$ sudo chmod -R 770 ./klass2 ./vol_mount

# Set the owner and group of the volume mount directories to 2222:2222
$ sudo chown -R 2222:2222 ./klass2 ./vol_mount

# Set the owner and group specifically for RabbitMQ which runs as UID 999:GID 999
$ sudo chown -R 999:999 ./vol_mount/db/rabbitmq_broker

Dotenv fileïƒ

The .env file stores the following variables, in the event that you need to create your own.

These are for the creation and accessing of the RabbitMQ message broker that handles task management and routing.

You can use any other password of your preference.

RABBIT_USER=admin
RABBIT_PWD=mypass

Import Docker Imageïƒ

Import the Docker image TAR archive klass2-dashboard.tar into your local Docker repository:

$ docker load -i klass2-dashboard.tar
$ docker images  # Check if the image is registered, you should see klass2-dashboard:latest
REPOSITORY             TAG            IMAGE ID       CREATED        SIZE
klass2-dashboard       latest         4bad4d5e8fe3   2 hours ago    9.17GB

Edit docker-compose.yml for Windows deploymentïƒ

If you are deploying on a Windows system, you will need to make this change in docker-compose.yml.

No change is required for Linux/Mac.

# change from this:
volumes:
    - ../vol_mount/db/rabbitmq_broker:/var/lib/rabbitmq/mnesia

# to this:
volumes:
    - ../vol_mount/db/rabbitmq_broker:/var/lib/rabbitmq

Launch the deploymentïƒ

In the ./klass2 directory, execute the following command to start the app:

$ docker-compose up -d --build

# to check that the containers have spun up
$ docker ps

To stop the app, use:

$ docker-compose down

Inference Method 2: Build your own Image (OPTIONAL)ïƒ

This method is mostly similar to Method 1, but instead, the ./klass2-final-deployment/klass2 directory will contain the source code, and docker-compose.yml will be inside it.

Unzip the file klass2-final-deployment-inference-source-code.zip and go to the directory containing the unzipped directory

Prepare vol_mount directory structureïƒ

Set up the vol_mount directory the same way as per Method 1, alongside the klass2 directory.

Also set the directory permissions as described in Method 1.

Launch the deploymentïƒ

In the klass2 directory, execute the following command to start the app. The --build tells Docker to build from source:

$ cd ./klass2-final-deployment/klass2
$ docker-compose up -d --build

To stop the app, use:

$ docker-compose down
